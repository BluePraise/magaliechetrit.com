How to make a search and filter with pure JavaScript

Posted on: 5 July 2022.

What exactly is it?

I was working on a WordPress portfolio project for a friend and she wanted a search filter on her site. I searched and tried out a few libraries. Although they were very good, they weren't what I was looking for. Most active search filters require a database connection or XHR requests. That was a bit too heavy for me. All I wanted was a functionality that searched through a dataset on the current page, something light and simple.

Time to make something myself. This is a search filter that updates dynamically while the user types. It's very light weight, it doesn't have any bells or whistles. Although you could certainly adapt it with a button or animations.

I've written this for newbies, so if you're advanced - you may get a little bored.

1. The search input.
In your html file, you can adapt your code to my example below or just copy and paste it. By the way, make sure that the classname is the same. ü´† 

<input type="search" class="js-search-field" placeholder="Type a word.."/>
More details about this concept

The input has a classname .js-search-field (in HTML). Then in JavaScript it has an EventListener attached to it. That eventlistener "scans" or "listens" for the users input from the keyboard. We will be using that information to check if the word can be found.

If it turns out that in your project you can't paste that input - you can change the classname to another classname or identifier you may have to use in the JavaScript.

2. The search items.
If you're a beginner this is where it may get tricky. But if you're using pure HTML and CSS like I am, then it's less comlicated. If we take a lot at the markup in the demo, we'll see that the list items we'd like to search through has a classname search-item. In your HTML file the dataset also needs to have that classname. If you cannot apply the classname, change the name in JavaScript. You will find an example below.

An example. 
const items = [...document.querySelectorAll('.any-name-that-applies')]
More details about this concept

In JavaScript we assemble an array of all of these search items. The input we've used at 1. (The search input) tries to search by using the filter() function with what the user inputs.

3. Putting it all together. When we go back to the script in the demo, we see another classname referenced: search-list. This is mainly a container, which holds our data-set. I've chosen to use an unordered list. While the user inputs a search term, this container is emptied and if it finds a match you will see those matches there.

The same advice stands, feel free to use this classname. Otherwise you need to change the classname in the JavaScript.

And that's it.
